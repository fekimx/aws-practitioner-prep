Introduction

Cloud Computing: On-demand delivery of IT resources and applications through the Internet with pay as you go pricing model.

Benefits of Cloud Computing are as follows:

  • Pay as you go
  • Don’t worry about capacity constraints
  • Low latency

Cloud Computing deployment models are as follows:

  • Cloud-based
  • On-premises/Private cloud
  • Hybrid

Amazon EC2 (Elastic Compute Cloud): A virtual server that provides scalable computing capacity.

Multitenancy: Sharing underlying hardware among multiple virtual machines (VMs).

ELB (Elastic Load Balancing): Automatically distributes incoming application traffic across multiple targets, such as Amazon EC2 instances, containers, and IP addresses. Works with auto-scaling groups to adjust capacity based on demand.

Loosely coupled architecture: Each service is independent (microservices approach), preventing cascading failures from a single point of failure. Queues help manage communication between services.

Tightly coupled components: All components are interconnected, making the system vulnerable to failures (monolithic application)

Application Integration

Amazon SQS (Simple Queue Service): Enables you to send, store, and receive messages between software components at any volume, facilitating decoupled communication.

Amazon SNS (Simple Notification Service): A pub/sub (publish/subscribe) messaging service that allows you to send notifications to multiple subscribers, such as web servers, email addresses, and AWS Lambda functions.

Amazon EventBridge: A serverless event bus service that facilitates the integration of applications using events. It allows for real-time event processing and helps decouple microservices, making it easier to build scalable and responsive applications.

AWS Step Functions: A serverless orchestration service that enables you to coordinate multiple AWS services into serverless workflows.

EC2

EC2 Instance Types

Key components are CPU, memory, storage, and networking.

  • General Purpose: Ideal for web servers and code repositories.
  • Compute Optimized: Best for gaming, scientific computing, HPC (High Performing Computing).
  • Memory Optimized: Suited for memory-intensive applications such as:
  • In-memory Databases (e.g., Redis or Memcached),
  • High-Performance Databases (e.g., Amazon Aurora),
  • Real-time Data Processing: Systems that require fast access to large datasets (e.g., streaming applications)
  • Accelerated Optimized: Designed for floating point calculations, graphics processing, data pattern matching.
  • Storage Optimized: Optimized for high storage throughput and capacity such as: log processing, backup and archiving, content distribution, data warehousing: Analytics platforms that require high storage throughput (e.g., Amazon Redshift).

EC2 Pricing Models:

  • On-demand Instances: Default purchasing option. Ideal for short-term, unpredictable workloads.
  • Reserved Instances: Long-term contracts (1 or 3 years) with upfront payment.
  • Standard: Cost-effective for steady-state workloads.
  • Convertible: Allows changing instance types during the term.
  • Spot Instances: A cost-effective option for running workloads that are flexible in terms of when they start and end, but they come with the risk of being interrupted. Available at up to 90% off the On-demand price. You specify the maximum. price you’re willing to pay. If your bid meets or exceeds the current Spot price, your instance runs. No long-term commitment; access to unused EC2 capacity. Ideal for applications such as batch processing, data analysis, or development/testing environments.
  • Dedicated Instances: Run in VPCs (Virtual Private Clouds) on hardware dedicated to a single customer. Provides isolation and compliance for sensitive workloads.

Amazon EC2 Fleet: A service that enables you to provision and manage a fleet of EC2 instances across different instance types and purchasing options (On-Demand, Spot, and Reserved Instances) to optimize for cost and performance.

AWS Free Tier and Support Plans

AWS Free Tier

The AWS Free Tier provides free access to a limited amount of AWS resources after signing up for an account. It includes three types of offers:

  1. Always Free: These offers are available to all AWS users and do not expire after 12 months. Examples include:
  • 1 million free requests per month for AWS Lambda.
  • 5 GB of Amazon S3 storage.
  1. 12-Month Free: These offers are available for 12 months after you sign up. Examples include:
  • 750 hours per month of t2.micro or t3.micro instances (EC2).
  • 5 GB of standard storage with Amazon RDS.
  1. Trials: These are short-term trials that provide free usage for a limited time, often for specific services. For example:
  • Free trials for Amazon Redshift or AWS Device Farm.

Factors to Consider in Cloud Services

  • Compliance: Ensure that the cloud provider meets industry standards and regulations relevant to your business (e.g., GDPR, HIPAA). Compliance is crucial for data protection and legal requirements.
  • Proximity/Latency: Choose a cloud region close to your users to reduce latency and improve application performance. Consider the geographical distribution of your user base and the impact on response times.
  • Feature Availability: Evaluate the features and services offered by the cloud provider. Ensure that the necessary tools, APIs, and services are available to meet your application needs.
  • Pricing: Analyze the pricing models and overall costs associated with the cloud services. Consider different pricing options (e.g., On-Demand, Reserved, Spot) to optimize costs based on your workload requirements.

AWS Infrastructure Overview

AWS Global Infrastructure: The overall architecture comprising Regions, AZs, Edge Locations, and Local Zones that enables redundancy and fault tolerance.

Regions: Geographically isolated areas that consist of 3 or more Availability Zones. E.g. us-east-1 (Northern VA)

Local Zones: Extensions of Regions that place compute, storage, and database services closer to end-users for applications that require low latency. E.g. us-east-1-ash-1 (Ashburn, VA)

AZ (Availability Zone): A single or a group of data centers within a Region. Each AZ is fully isolated from others, ensuring fault tolerance and high availability within the AWS infrastructure. E.g. us-east-1a

Edge Locations: A site that CloudFront uses to cache copies. Part of the AWS global network that delivers content closer to users, primarily used for services like Amazon CloudFront. E.g. IAD (Dulles, VA)

AWS Wavelength: A service that brings AWS services to the edge of the 5G network, enabling low-latency applications. E.g. wavelength-us-east-1

Storage Services

Amazon EBS (Elastic Block Storage):

A virtual hard drive for use with EC2 instances, providing block-level storage.  

Key Features:

  • Data is stored in volumes as blocks.
  • Does not automatically scale.
  • Typically used for applications requiring low-latency access to data.
  • EBS and EC2 must reside in the same AZ.
  • EBS snapshots back up data incrementally.

Amazon EFS (Elastic File System): 

A scalable file storage service for use with AWS Cloud services and on-premises resources.

Key Features:

  • Regional service
  • Automatically scales based on storage needs.
  • Stores data in and across multiple AZs for high availability and durability.

Amazon S3 (Simple Storage Service):

A scalable object storage service designed for storing and retrieving any amount of data. Each object must be less than 5TB of data.

Key Features:

  • Data is stored as objects in buckets, making it suitable for a wide range of use cases, including backups, big data analytics, and static website hosting.
  • Accessed over HTTP/HTTPS, ideal for high durability and availability but with higher latency compared to EBS.

Bucket versioning: It gives you the ability to maintain multiple versions of an object in an S3 bucket. Therefore, if you accidentally delete or overwrite an object, Amazon S3 does not remove the object permanently. Instead, Amazon S3 inserts a delete marker for the object in buckets with versioning turned on. This feature gives you the ability to restore previous versions of every object that is stored in the bucket.

Amazon S3 Acceleration: A feature of Amazon S3 that accelerates uploads and downloads for large files over long distances.

S3 Types:

  • S3 Standard: High durability, availability, and performance for frequently accessed data. Use Cases: Websites, mobile applications, data analytics, and content distribution. Retrieval Time: Milliseconds. The data is stored in at least 3 AZs.

  • S3 Intelligent-Tiering: Automatically moves data between two access tiers (frequent and infrequent) based on changing access patterns. Use Cases: Data with unpredictable access patterns, such as analytics and archives. Retrieval Time: Milliseconds. If an object is not accessed for 30 days, it is moved from Standard to Standard-IA.

  • S3 Standard-IA (Infrequent Access): Lower-cost option for data that is infrequently accessed but requires rapid retrieval/high availability. Use Cases: Backup data, disaster recovery, and long-term storage of data not accessed regularly. Retrieval Time: Milliseconds.

  • S3 One Zone-IA: Lower-cost option for infrequently accessed data that does not require multiple AZ resilience. Use Cases: Storing secondary backup copies, or data that can be recreated easily. Retrieval Time: Milliseconds.

  • S3 Glacier Flexible Retrieval: Low-cost storage for data archiving and long-term backup with retrieval times ranging from minutes to hours. Use Cases: Archiving data, compliance data retention, and digital media archives. Retrieval Time: Minutes to hours (depends on retrieval option)

  • S3 Glacier Deep Archive: The lowest-cost storage class designed for long-term data archiving. Use Cases: Long-term archiving for compliance or data retention needs. Retrieval Time: 12 Hours

AWS Storage Gateway: A hybrid cloud storage service that enables on-premises applications to seamlessly use AWS cloud storage.

AWS Backup: A centralized backup service that automates the backup of AWS services and on-premises resources.

AWS Elastic Disaster Recovery (DRS): A service designed to minimize downtime and data loss for applications running on AWS. It enables quick recovery of applications in case of a failure.

Security, Identity, and Compliance

AWS IAM (Identity and Access Management): A service that enables you to manage access to AWS resources securely. You can create and manage AWS users and groups, and use permissions to allow or deny their access to AWS resources.

Key Concepts:

  • Root User: The initial account created, has full access to all resources.
  • User: An individual with specific permissions. Users can be assigned roles and policies.
  • Policy: A document (in JSON format) that defines permissions; grants or denies actions on resources.
  • Groups: Collections of users that share the same permissions.
  • Roles: Temporary access permissions that can be assumed by users or AWS services.
  • MFA (Multi-Factor Authentication): An additional security layer requiring two or more verification methods for user access.

AWS Artifact: A service that provides on-demand access to AWS compliance reports and allows you to manage agreements with AWS.

Sub-services:

  • Agreements: Manage and review online agreements, such as the AWS Customer Agreement and Business Associate Addendum.
  • Reports: Access a variety of compliance reports, including SOC, ISO, PCI, and others, to ensure your compliance posture.

AWS Trusted Advisor 

A management tool that provides real-time guidance to help you reduce costs, increase performance, and improve security. It offers insights across five categories:

Cost Optimization: Identifies underutilized or idle resources to help reduce costs.

Performance: Provides recommendations to improve the performance of your AWS resources.

Security: Highlights security best practices, such as enabling MFA and reviewing access keys.

Fault Tolerance: Suggests ways to improve the resilience of your applications.

Service Limits: Monitors usage against service limits to prevent resource constraints.

Some examples are as follows:

  • Recommending the right instance types for EC2 based on usage patterns.
  • Identifying S3 buckets that don’t have public access settings configured properly.
  • Alerting on unused Elastic Load Balancers that can be removed to save costs.

AWS Well-Architected Framework

A set of best practices designed to help cloud architects build secure, high-performing, resilient, and efficient infrastructure for applications.

  1. Operational Excellence: Achieve effective workload management, gain valuable insights, and continuously enhance supporting processes. (Perform operations as code)
  2. Security: Encompasses the protection of data, systems, and assets while delivering business value. (Monitor, alert, audit actions and changes)
  3. Reliability: Consistently and accurately perform intended functions.
  4. Performance Efficiency: Utilize computing resources effectively to meet system requirements. (Deploy globally in minutes)
  5. Cost Optimization: Focuses on avoiding unnecessary costs and maximizing return on investment. (Analyze and attribute expenditures)
  6. Sustainability: Optimize your workloads to minimize their environmental impact while ensuring efficient resource use and operational resilience.

AWS Well-Architected Tool: A service that allows you to review and improve your workloads based on the best practices outlined in the Well-Architected Framework.

AWS CAF (Cloud Adoption Framework)

A structured approach that provides guidance across multiple dimensions to ensure a comprehensive and effective cloud migration and utilization. There are 6 key perspectives:

Business Perspective: Aligns cloud adoption with business goals, helping stakeholders understand benefits and ROI. (e.g. Product management)

People Perspective: Focuses on organizational change management, training, and skill development to support cloud adoption.

Governance Perspective: Establishes policies, controls, and compliance frameworks to manage risk and ensure governance in the cloud. (e.g. Application portfolio management)

Platform Perspective: Involves the technical aspects, such as architecture and integration, to optimize performance and security.

Security Perspective: Emphasizes the importance of security best practices and compliance throughout the cloud journey. (e.g. IAM)

Operations Perspective: Ensures operational readiness, including monitoring, incident response, and ongoing management of cloud resources.

Some examples are as follows:

  • Assessing current IT capabilities and identifying gaps in skills for effective cloud management.
  • Developing a roadmap for cloud migration that aligns with business objectives.
  • Implementing governance frameworks to ensure compliance with industry regulations.
  • Performance and capacity management

Cloud Migration Strategies (6 Rs)

As its name suggests, it refers to the various approaches organizations can take to move applications, data, and workloads from on-premises environments to the cloud. Here are the primary strategies:

  1. Rehosting (Lift and Shift): Move applications to the cloud with minimal changes, essentially “lifting” them from on-premises and “shifting” them to the cloud. Use Case: Quick migration of legacy applications to reduce on-premises infrastructure costs.
  2. Replatforming (Lift, Tinker, and Shift): Make a few optimizations to the application to leverage cloud services while avoiding major changes to the core architecture. Use Case: Migrating a database to Amazon RDS while keeping the application largely intact.
  3. Refactoring (Re-architecting): Redesign the application to fully utilize cloud-native features and services, often resulting in better performance and scalability. Use Case: Breaking a monolithic application into microservices for improved agility and scalability.
  4. Repurchasing: Move to a different product, often a SaaS solution, effectively replacing the existing application. Use Case: Switching from an on-premises CRM to a cloud-based CRM solution like Salesforce.
  5. Retiring: Identify and decommission applications that are no longer useful or needed. Use Case: Phasing out obsolete applications to streamline operations and reduce costs.
  6. Retaining: Keep certain applications on-premises or in their current state while determining their long-term cloud strategy. Use Case: Maintaining regulatory compliance for specific applications that cannot move to the cloud yet.

Shared Controls: Controls which apply to both the infrastructure layer and customer layers, but in completely separate contexts or perspectives. In a shared control, AWS provides the requirements for the infrastructure and the customer must provide their own control implementation within their use of AWS services. Examples include:

  • Patch Management – AWS is responsible for patching and fixing flaws within the infrastructure, but customers are responsible for patching their guest OS and applications.
  • Configuration Management – AWS maintains the configuration of its infrastructure devices, but a customer is responsible for configuring their own guest operating systems, databases, and applications.
  • Awareness & Training - AWS trains AWS employees, but a customer must train their own employees.

AWS Financial Management Tools

AWS Pricing Calculator: A tool that creates cost estimates.

AWS Billing Conductor: A flexible billing and pricing management tool that allows organizations to customize their AWS billing and usage reporting. It provides a way to create custom pricing plans, manage costs, and gain insights into AWS service usage, making it easier for businesses to allocate expenses accurately.

AWS Billing Dashboard: A central place to view your AWS billing and account usage. It allows you to compare billing periods.

AWS Budgets: A service that enables you to set custom cost and usage budgets for your AWS account and receive alerts when your usage exceeds defined thresholds. It is updated 3 times a day.

AWS Budgets Reports: Automates the generation and delivery of budget reports to keep stakeholders informed about spending.

AWS Cost Explorer: A tool that allows you to visualize and analyze your AWS spending patterns over time. It helps you understand your cost and usage trends.

AWS Cost and Usage Report: It tracks your AWS usage and provides estimated charges associated with your account.

AWS Marketplace: An online store that provides a curated catalog of software, data, and services. It allows customers to find, buy, and immediately deploy third-party applications and services.
