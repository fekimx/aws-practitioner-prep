Security, Identity, and Compliance

AWS IAM (Identity and Access Management): A service that allows you to create and manage AWS users and groups, and use permissions to allow or deny their access to AWS resources.

Root User: The initial account created, has full access to all resources.

User: An individual with specific permissions. They can be assigned roles and policies.

Policy: A document (in JSON format) that defines permissions; grants or denies actions on resources.

Groups: Collections of users that share the same permissions.

Roles: Temporary access permissions that can be assumed by users or AWS services.

MFA (Multi-Factor Authentication): An additional security layer requiring two or more verification methods for user access.

AWS Artifact: A service that provides on-demand access to AWS compliance reports and allows you to manage agreements with AWS.

AWS Infrastructure Overview

AWS Global Infrastructure: The overall architecture comprising Regions, AZs, Edge Locations, and Local Zones that enables redundancy and fault tolerance.

Regions: Geographically isolated areas that consist of 3 or more Availability Zones.

Local Zones: Extensions of Regions that place compute, storage, and database services closer to end-users for applications that require low latency.

Availability Zone: A single or a group of data centers within a Region.

Edge Locations: A site that CloudFront uses to cache copies.

AWS Wavelength: A service that brings AWS services to the edge of the 5G network, enabling low-latency applications.
